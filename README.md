. Introduction
1.1 Purpose: Define the requirements for managing customer information in the banking system.
1.2 Scope: Handles customer onboarding, profile updates, and secure storage.
2. Overall Description
2.1 User Classes: Bank Customers (view/update), Bank Staff (create/update), Administrator.
2.2 Operating Environment: Web browser or Mobile App with backend database (e.g., MySQL).
3. Functional Requirements (Data Fields)
3.1 Customer Registration: Capture Name, Address, Phone, Email, DOB, Gender, KYC documents.
3.2 Account Details: Account Number (unique), Account Type, Branch Code, Account Status.
3.3 Authentication: Secure Login (Username/Password), PIN, OTP verification.
3.4 Profile Management: Customer can update address/contact details, subject to verification.
4. External Interface Requirements
4.1 User Interface: Web-based GUI (HTML/PHP), login screen.
4.2 Hardware/Software Interfaces: Connection to central banking database.
5. Non-Functional Requirements
5.1 Security: Data encryption (SSL/HTTPS), password hashing, role-based access.
5.2 Reliability: 99.9% uptime, data integrity during transactions.
5.3 Performance: Fast retrieval of customer profile data.
6. Business Rules
Unique PAN/Account number validation.
Mandatory fields must not be empty.
Minimum age requirements for account opening.
